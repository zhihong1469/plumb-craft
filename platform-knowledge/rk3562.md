## 一、专属约束

### 1.1 编译器要求

| 项目 | 要求 |
|------|------|
| 工具链 | `arm-linux-gnueabihf-gcc` |
| 最低版本 | gcc 9.0+ |
| 架构 | ARM Cortex-A35 (ARMv8-A) |

### 1.2 内核版本

| 项目 | 要求 |
|------|------|
| 推荐版本 | Linux 5.10 |
| 设备树路径 | `/boot/dtb/rk3562-*.dtb` |
| 设备树名称 | `rk3562-evb1-v10.dtb`（通用） |

### 1.3 内存布局

| 项目 | 说明 |
|------|------|
| DDR容量 | 1GB/2GB/4GB |
| 起始地址 | `0x00000000` |
| 内核地址 | `0x00200000` |
| DMA预留 | `0x10000000-0x40000000` |

## 二、硬件加速能力

### 2.1 硬件加速模块概览

| 模块 | 名称 | 功能 | 性能指标 |
|------|------|------|---------|
| **VPU** | Video Processing Unit | 视频编解码 | H.264/H.265 4K@30fps |
| **RGA** | Raster Graphic Accelerator | 图像预处理 | 格式转换、缩放、旋转 |
| **NPU** | Neural Processing Unit | AI推理 | 0.5 TOPS INT8 |
| **GPU** | Mali-G52 | 图形渲染 | OpenGLES 3.2 |

### 2.2 VPU（视频处理单元）

#### 支持格式

| 编码格式 | 最大分辨率 | 帧率 |
|---------|-----------|------|
| H.264 | 4K (3840x2160) | 30fps |
| H.265 | 4K (3840x2160) | 30fps |
| VP9 | 4K (3840x2160) | 30fps |

#### 驱动与库

```bash
# 驱动模块
modprobe rockchip-vpu
modprobe rockchip-vpu-h264
modprobe rockchip-vpu-h265

# 多媒体处理库 (RK MPP)
# 路径: /usr/lib/librockchip_mpp.so
```

#### RK MPP API 速查

```c
// 初始化 MPP 上下文
MPP_RET mpp_create(&ctx, &api);
MPP_RET mpp_init(ctx, MPP_CTX_DEC, MPP_CODEC_H265);

// 设置输入输出
MPP_RET mpp_dec_set_frame_buffer_planes(ctx, &fb);
MPP_RET mpp_dec_set_input(ctx, &packet);

// 解码循环
MPP_RET mpp_dec_decode(ctx, &packet, &frame);
```

### 2.3 RGA（光栅图形加速器）

#### 支持操作

| 操作类型 | 说明 | 性能 |
|---------|------|------|
| 格式转换 | YUV/RGB 互转 | 300MP/s |
| 缩放 | 图像缩放 | 200MP/s |
| 旋转 | 0/90/180/270度 | 150MP/s |
| 裁剪 | 区域裁剪 | 200MP/s |
| 混合 | Alpha混合 | 100MP/s |

#### librga API 速查

```c
#include <rga.h>

// 初始化
rga_buffer_t src, dst;
rga_init();

// 设置源缓冲区
src.width = 1920;
src.height = 1080;
src.format = RK_FORMAT_YCbCr_420_SP;
src.virAddr = src_buffer;

// 设置目标缓冲区
dst.width = 640;
dst.height = 480;
dst.format = RK_FORMAT_RGB_888;
dst.virAddr = dst_buffer;

// 执行转换
rga_blit(&src, &dst, NULL);
```

#### 性能对比

| 操作 | 软件实现 | RGA硬件加速 | 提升倍数 |
|------|---------|------------|---------|
| 1080p YUV→RGB | ~50ms | ~5ms | 10x |
| 4K缩放(1/4) | ~100ms | ~8ms | 12x |

### 2.4 NPU（神经网络处理器）

#### 硬件规格

| 项目 | 规格 |
|------|------|
| 算力 | 0.5 TOPS (INT8) |
| 支持模型 | Caffe/TensorFlow/TFLite/ONNX |
| 量化支持 | INT8/INT16/Float16 |

#### RKNN API 速查

```python
from rknn.api import RKNN

# 创建 RKNN 对象
rknn = RKNN()

# 加载模型
rknn.load_tflite(model='model.tflite')

# 构建模型
rknn.build(do_quantization=True, dataset='dataset.txt')

# 导出 RKNN 模型
rknn.export_rknn('model.rknn')

# 初始化运行时
rknn.init_runtime()

# 推理
outputs = rknn.inference(inputs=[input_data])

# 释放资源
rknn.release()
```

#### 性能对比（MobileNet v2）

| 平台 | 推理时间 | FPS |
|------|---------|-----|
| CPU (A35) | ~200ms | 5 |
| NPU 加速 | ~25ms | 40 |
| 提升倍数 | - | **8x** |

## 三、交叉编译配置

### 3.1 工具链设置

```bash
# 环境变量配置
export CROSS_COMPILE=arm-linux-gnueabihf-
export CC=${CROSS_COMPILE}gcc
export CXX=${CROSS_COMPILE}g++
export LD=${CROSS_COMPILE}ld
export AR=${CROSS_COMPILE}ar
export STRIP=${CROSS_COMPILE}strip

# 目标架构
export ARCH=arm
export TARGET=arm-linux-gnueabihf
```

### 3.2 编译选项

```bash
# 基础编译选项
CFLAGS="-march=armv8-a -mtune=cortex-a35 -mfpu=neon-fp-armv8"
CFLAGS+=" -O2 -pipe"
CFLAGS+=" -g"

# RK MPP 库路径
CFLAGS+=" -I/path/to/mpp/include"
LDFLAGS+=" -L/path/to/mpp/lib -lrockchip_mpp"

# librga 库路径
CFLAGS+=" -I/path/to/rga/include"
LDFLAGS+=" -L/path/to/rga/lib -lrga"

# RKNN 库路径
CFLAGS+=" -I/path/to/rknn/include"
LDFLAGS+=" -L/path/to/rknn/lib -lrknn_api"
```

### 3.3 硬件加速库获取

| 库名称 | 获取方式 | 版本要求 |
|--------|---------|---------|
| RK MPP | Rockchip SDK | v1.4+ |
| librga | Rockchip SDK | v2.0+ |
| RKNN Toolkit | GitHub release | v1.7+ |

## 六、启动流程

### 6.1 启动顺序

1. **BootROM**: 芯片内部固件
2. **TPL (Third Program Loader)**: 初始化 DDR
3. **SPL (Secondary Program Loader)**: 加载 U-Boot
4. **U-Boot**: 加载内核和设备树
5. **Kernel**: 启动 Linux 内核

### 6.2 启动参数

```bash
setenv bootargs 'console=ttyS2,1500000n8 root=/dev/mmcblk0p2 rootfstype=ext4 rw rootwait'
setenv bootcmd 'mmc read 0x40008000 0x4000 0x8000; bootm 0x40008000'
saveenv
```

## 七、性能优化建议

### 7.1 内存优化

| 优化项 | 建议 | 收益 |
|--------|------|------|
| DMA缓冲区 | 使用物理连续内存 | 减少内存拷贝 |
| 缓存策略 | 启用 L2 cache | 提升数据访问速度 |
| 内存映射 | 使用 mmap | 避免数据拷贝 |

### 7.2 硬件加速使用建议

| 场景 | 推荐方案 | 性能提升 |
|------|---------|---------|
| 视频解码 | RK MPP VPU | 10x+ |
| 图像预处理 | librga | 8-12x |
| AI推理 | RKNN NPU | 8x+ |
| 图形渲染 | Mali-G52 GPU | 5x+ |

## 八、常见问题

### 8.1 编译错误

```bash
# 错误: cannot find -lrockchip_mpp
# 解决: 添加 MPP 库路径
LDFLAGS+=" -L/path/to/mpp/lib"

# 错误: undefined reference to rga_blit
# 解决: 添加 librga
LDFLAGS+=" -lrga"

# 错误: undefined reference to rknn_init
# 解决: 添加 RKNN 库路径和链接库
LDFLAGS+=" -L/path/to/rknn/lib -lrknn_api"
```

### 8.2 运行时错误

```bash
# 错误: error while loading shared libraries: librga.so
# 解决: 设置库路径
export LD_LIBRARY_PATH=/path/to/rga/lib:$LD_LIBRARY_PATH

# 错误: NPU not found
# 解决: 检查设备树中 NPU 节点状态

# 错误: MPP init failed
# 解决: 确保加载了 VPU 驱动模块
modprobe rockchip-vpu
modprobe rockchip-vpu-h264
```

### 8.3 性能问题

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 帧率低 | CPU 占用过高 | 启用硬件加速，检查是否使用了软件实现 |
| 花屏 | 格式不匹配 | 检查输入输出格式是否正确配置 |
| 内存不足 | 缓冲区分配过大 | 优化缓冲区大小，使用 DMA 内存 |
