# 整体分层架构

> 六层线束架构 ↔ 大模型完整思考链路

---

## 架构分层总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        六层线束架构 ↔ 大模型思考链路                     │
├─────────────────────────────────────────────────────────────────────────┤
│  L6 安全护栏层    → 这么做有没有风险？会不会违规？（首尾双放置 + 每步校验）│
│  L5 输出模板层    → 输出结果该按什么格式来？（生成前调用）                 │
│  L4 专项技能层    → 具体步骤是什么？该调用什么工具？（按需激活）           │
│  L3 平台知识层    → 这个平台/芯片有什么特殊要求？（按需加载）             │
│  L2 通用规则层    → 做这件事要遵守什么通用规矩？（全局常驻）               │
│  L1 技能索引层    → 这是什么任务？该用哪个技能？（全局常驻）               │
│  L0 身份边界层    → 我是谁？我能做什么？不能做什么？（全局常驻）           │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 各层详细设计

### L0 身份边界层

**职责**：定义角色、核心能力、禁止事项

**加载时机**：全局常驻（System Prompt 最前端）

**设计要点**：
- 极简，抢占最高注意力权重
- 明确角色定位
- 列出核心能力清单
- 明确禁止事项

**物理位置**：`meta-spec/00_principles.md`

**示例内容**：
```markdown
# 身份边界

你是一个嵌入式AI技能助手，专注于嵌入式开发任务。

核心能力：
- 编译构建
- 烧录调试
- 代码生成
- 问题诊断

禁止事项：
- 不执行高危操作（如格式化磁盘）
- 不访问用户隐私数据
- 不修改系统关键配置
```

---

### L1 技能索引层

**职责**：技能清单、触发关键词、入口路径

**加载时机**：全局常驻（紧随身份层）

**设计要点**：
- 表格化，快速匹配分发
- 包含所有可用技能
- 明确触发关键词
- 提供入口路径

**物理位置**：`meta-spec/01_skills_index.md`

**示例内容**：
```markdown
# 技能索引

| 技能名称 | 触发关键词 | 入口路径 | 平台支持 |
|---------|-----------|---------|---------|
| build-linux-app | 编译,build,make,cmake | skills/build-linux-app/SKILL.md | Linux |
| flash-imx6ull | 烧录,flash,刷机 | skills/flash-imx6ull/SKILL.md | i.MX6ULL |
```

---

### L2 通用规则层

**职责**：编码规范、脚本标准、依赖要求

**加载时机**：全局常驻（索引之后）

**设计要点**：
- 基础约束前置
- 所有任务默认生效
- 提供编码规范
- 定义脚本标准

**物理位置**：`meta-spec/02_general_rules.md`

**示例内容**：
```markdown
# 通用规则

编码规范：
- 使用Python 3.8+
- 遵循PEP 8规范
- 添加类型注解
- 编写单元测试

脚本标准：
- 统一错误处理
- 结构化输出
- 跨平台兼容
- 日志记录
```

---

### L3 平台知识层

**职责**：芯片/平台专属约束、API速查、编译参数

**加载时机**：按需激活（匹配硬件任务时）

**设计要点**：
- 仅涉及硬件时注入
- 平台专属约束
- API速查表
- 编译参数参考

**物理位置**：`platform-knowledge/*.md`

**示例内容**：
```markdown
# RK3562 平台知识

专属约束：
- 必须使用 arm-linux-gnueabihf-gcc 编译
- 内核版本：5.10
- 设备树路径：/boot/dtb/rk3562-xxx.dtb

API速查：
- GPIO操作：/sys/class/gpio/
- 串口设备：/dev/ttySx
- I2C设备：/dev/i2c-x
```

---

### L4 专项技能层

**职责**：具体步骤、工具调用、业务逻辑

**加载时机**：按需激活（匹配对应技能时）

**设计要点**：
- 任务确认后加载完整细节
- 单一职责
- 标准化输出
- 完整的测试用例

**物理位置**：`skills/*/SKILL.md`

**示例内容**：
```markdown
---
name: build-linux-app
version: 1.0.0
description: 编译 Linux 应用程序
keywords: ["编译", "build", "make", "cmake"]
---

# 技能说明

## 触发条件
- 用户提到"编译"、"build"、"make"等关键词
- 当前目录存在 Makefile 或 CMakeLists.txt

## 执行步骤
1. 检测工具链
2. 读取构建配置
3. 执行构建命令
4. 返回结构化结果

## 输出格式
| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | success/partial/failure |
| summary | string | 执行摘要 |
| evidence | array | 输出文件列表 |
```

---

### L5 输出模板层

**职责**：输出格式、结构定义、返回标准

**加载时机**：生成前调用（仅输出阶段）

**设计要点**：
- 贴合结尾强注意力
- 标准化输出格式
- 支持多种输出类型
- 可扩展模板

**物理位置**：`templates/*/*.md`

**示例内容**：
```markdown
# 代码输出模板

```python
# 自动生成的代码
# 生成时间：{timestamp}
# 技能版本：{skill_version}

{code_content}
```

## 输出说明
- 代码遵循PEP 8规范
- 添加了必要的注释
- 包含类型注解
```

---

### L6 安全护栏层

**职责**：高危操作清单、人工确认机制、权限分级

**加载时机**：全局常驻（首尾双放置）+ 工具调用前校验

**设计要点**：
- 三重保险：开头定调、结尾兜底、调用前拦截
- 高危操作清单
- 人工确认机制
- 权限分级

**物理位置**：`meta-spec/06_security_rule.md`

**示例内容**：
```markdown
# 安全红线

高危操作清单：
| 操作类型 | 风险等级 | 处理策略 |
|----------|---------|---------|
| 硬件烧录 | 高 | 必须人工确认 |
| 系统格式化 | 高 | 必须人工确认 |
| 网络请求 | 中 | 白名单控制 |
| 文件覆盖 | 中 | 备份机制 |

三重校验：
1. 用户请求 → L6 前置检查
2. 工具调用前拦截
3. 执行后审计
```

## 渐进式信息揭露

这是本架构和普通技能集最大的差异：

### 普通技能集
- 把所有规则、所有技能说明全塞进上下文
- 大模型自己翻，翻到多少算多少
- 冗余信息严重干扰注意力

### 本架构
1. **常驻层（L0/L1/L2/L6首尾）极简**：只放最核心的底线规则，占token极少
2. **扩展层（L3/L4/L5）按需加载**：
   - 用到硬件才加载平台知识
   - 确定用哪个技能才加载技能详情
   - 要输出了才给模板
3. **完全顺着大模型的思考节奏**：需要什么给什么，永远不让无关信息占用注意力

### 类比
这就像 **按需分页 + 动态加载** 的内存管理机制：
- 常驻内核极小
- 用户态模块按需调入内存
- 不用的就换出
- 既保证了系统稳定，又最大化了运行效率

## 总结

| 层级 | 名称 | 加载时机 | 设计要点 |
|------|------|----------|---------|
| L0 | 身份边界层 | 全局常驻 | 极简，抢占最高注意力权重 |
| L1 | 技能索引层 | 全局常驻 | 表格化，快速匹配分发 |
| L2 | 通用规则层 | 全局常驻 | 基础约束前置 |
| L3 | 平台知识层 | 按需激活 | 仅涉及硬件时注入 |
| L4 | 专项技能层 | 按需激活 | 任务确认后加载完整细节 |
| L5 | 输出模板层 | 生成前调用 | 贴合结尾强注意力 |
| L6 | 安全护栏层 | 全局常驻+调用前校验 | 三重保险 |
